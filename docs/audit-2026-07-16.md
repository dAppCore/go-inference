## (a) Unreferenced / fake-coverage tests — FIXED this pass (79)

`unreferenced-symbols.py` flags `Test<File>_<Symbol>_<Variant>` functions whose body
never names the `<Symbol>` slot — the strongest available signal for "claims a symbol,
never exercises it." All 79 offenders inside `go/model/**` and `go/serving/**` turned
out to be **real, working tests with a naming defect**, not empty theatre — each one
already drove genuine assertions against genuine production behaviour; the checker
just couldn't find its own claimed target as a literal token in the body. Four
(`ernie45`/`exaone4`/`glm4`/`hunyuan` `Config_Arch_Ugly`) were a real gap: all four
copy-pasted the same `Ugly` case testing `Parse()`'s malformed-JSON handling instead of
`Arch()`'s own edge case, leaving `Arch()`'s GQA head-divisibility guard genuinely
unexercised by any Ugly variant.

Fix shape applied per finding class (never by deleting a test or gaming an assertion):

- **Naming-only mismatch (74 of 79)** — the test's symbol slot named a synthetic
  concept (`MistralQuantizeLane`, `WeightIndex`, `MethodRejection`) that isn't a real Go
  identifier, while the body already called something real and central
  (`NewTransformerQuantizeLane`, `WeightMap`, `http.StatusMethodNotAllowed`,
  `wrapWelfare`, `ResolveModel`, …). Renamed the symbol slot to the real, already-present
  identifier (or appended it as a second underscore-joined segment, preserving the
  original descriptive word — house convention already accepts multi-word compounds,
  e.g. `ClassifyBatch_ApplyPreset`). Zero behavioural change; every rename verified by
  re-running `unreferenced-symbols.py` afterwards (430→351, exactly −79, zero
  regressions, zero leakage outside `model/`+`serving/`).
- **Real logic gap (4 of 79)** — `ernie45`/`exaone4`/`glm4`/`hunyuan` `Config_Arch_Ugly`:
  rewrote each to Parse a syntactically-valid, otherwise-complete config whose
  `NumAttentionHeads % NumKeyValueHeads != 0` (the shared GQA guard visible in all four
  `Arch()` implementations), asserting `Arch()` itself rejects it — a genuine edge case
  distinct from `_Bad`'s all-zero-fields rejection. `hunyuan` got its own arch-specific
  edge instead (`use_cla:true` + `cla_share_factor:0`, the CLA-share-factor guard) since
  it's more distinctive than reusing the shared head-divisibility check a third time.
  All four now reference `Arch()` directly (`c.Arch()`), so no rename was even needed —
  the body fix resolved the checker independently.
- **Delegate-only body (1 of 79)** — `mistral`'s `_Good` variant only called a private
  helper (`testMistralLane`) that itself calls `NewTransformerQuantizeLane`, so the
  literal call never appeared in the `_Good` test's own body (unlike its `_Bad`/`_Ugly`
  siblings, and unlike `phi3`/`qwen3`'s inlined `_Good`). Added one direct, real
  `Detect()` assertion (previously only exercised via `_Ugly`'s malformed-input case,
  now also proven on a valid `mistral3` config) ahead of the existing delegate calls.

Full list of the 79 fixes (old name → new name, `file:line`): see the commit diff on
`chore/cga-audit` — each is a one-line function-signature change (plus doc-comment
sync) except the five body edits called out above.

**Methodology caveat (evidenced, not asserted)**: Go's default per-package coverage
mode *understates* library functions exercised primarily through OTHER packages'
tests. Proof: `model/gguf/transformer_lane.go:44 NewTransformerQuantizeLane` shows
0.0% in this sweep — yet it's the exact function this pass's cluster-(a) fix proved
live via 9 real tests across `model/arch/{mistralai/mistral,microsoft/phi,Qwen/qwen3}/gguf`.
Those tests live in consumer packages' own test binaries, so `model/gguf`'s own
instrumented coverage never sees the calls. **Before treating any of the 132 above as
"needs a test," grep for cross-package callers first** — a meaningful fraction are
likely the same shape (real, exercised, just not from within their own package's test
binary). `model/composed` and `model/arch/bert` are worth the coverage-unit treatment
regardless (they double up with the AX-7/Example gaps above), but confirm caller
status symbol-by-symbol before assuming "untested."

consumer repos migrating onto core/go v0.9.0) is not part of this audit's scope, and
most of it reads as false-positive noise for an engine repo with its own stdlib-shaped

`TextModel`/`Backend` deliberately use idiomatic `(T, error)` and `iter.Seq[Token]` +

dimension worth flagging on its own merits regardless of the v0.9.0 framing:

- **`banned-imports` = 738** hits across production + test files (`fmt`/`errors`/`strings`/`os`/`log`/`json`/`bytes` instead of `core.E` / `c.Strings()` / `c.Fs()` etc). Whether this is a live house-rule violation or a sanctioned exception for an engine repo (perf-critical paths, stdlib-interface implementations that must accept `[]byte`/`string` on the wire) needs an operator call, not agent action — flagging per the lethean-perf skill's own caveat ("idiom reds need context before agents").
- `licence-missing`, `docs-gaps` (6), `tracked-artifacts` (5) are quick, mechanical, low-risk fixes if ever in scope — not investigated further here since they're outside the seven requested clusters.
